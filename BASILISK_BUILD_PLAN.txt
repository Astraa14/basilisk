OVERVIEW
--------
Transform Basilisk from local CLI tool into a complete SaaS platform where users:
1. Install the tool with one command

TASK 1.3: Create Device Auth Module
------------------------------------
File: basilisk/auth.py

Create a new module that handles browser-based login using device code flow 
(like GitHub CLI auth). This way users don't type passwords in terminal. It should:
- Request a device code from the backend
- Get back a user code (like ABC-DEF-GHI) and verification URL
- Open user's default browser to that URL
- Poll the backend repeatedly to check if user confirmed in browser
- When confirmed, receive and return the backend API key
- Timeout after 60 seconds if user doesn't confirm

Functions to create:
- request_device_code() - calls backend POST /api/auth/device-code, 
  returns device_code, user_code, verification_uri
- poll_for_backend_key(device_code) - repeatedly calls backend POST /api/auth/token 
  until user confirms, returns api_key
- open_auth_browser(url) - opens URL in default browser using webbrowser module

Flow:
1. User runs `basilisk auth`
2. CLI calls request_device_code()
3. CLI prints the URL and code
4. CLI calls open_auth_browser() to open browser
5. CLI shows loading spinner and calls poll_for_backend_key()
6. When user confirms in browser, returns api_key
7. CLI saves api_key with config.save_backend_api_key()

Dependencies needed:
- import webbrowser
- import requests
- import time

TASK 2.6: Create Auth Service
-----------------------------
File: basilisk-backend/auth.py

Create functions to handle device code flow authentication:

1. generate_device_code() function:
   - Create random device_code string (32 chars using secrets module)
   - Create random user_code string (6 chars like ABC-DEF-GHI)
   - Create verification_uri URL pointing to frontend auth page
   - Store in Redis (or in-memory dict) with 10-minute expiry
   - Mark as not-yet-verified in storage
   - Return all three values
   - You can use a simple dict with datetime expiry for now

2. verify_device_code(user_code, user_id) function:
   - Look up the device code by user_code in storage
   - Mark it as verified=True
   - Link it to user_id
   - Return True if found and updated
   - Return False if not found

3. get_verified_token(device_code) function:
   - Check if device_code exists in storage
   - Check if it's marked verified=True
   - If yes: generate new API key, save to User.api_key in database
   - Delete the device code entry from storage (one-time use)
   - Return the API key
   - If not verified yet: return None

4. generate_api_key() function:
   - Create random 32-character string using secrets.token_urlsafe()
   - Prefix with "bsk_" so it's clearly a Basilisk key
   - Return the key

5. get_user_by_api_key(api_key, db) function:
   - Query database for User with matching api_key
   - Return User object or None

6. cleanup_expired_codes() function (optional):
   - Remove expired device codes from storage
   - Can run periodically or on each request


TASK 2.7: Create FastAPI Main App
---------------------------------
File: basilisk-backend/main.py

Create the FastAPI application with these endpoints:

AUTHENTICATION ENDPOINTS:

1. POST /api/auth/device-code
   - No request body needed
   - Returns DeviceCodeSchema (device_code, user_code, verification_uri, expires_in)
   - Calls generate_device_code() from auth module
   - Stores device code in storage
   - HTTP 200 OK

2. POST /api/auth/verify
   - Request body: VerifyDeviceSchema (user_code, username, email)
   - Returns {"verified": true}
   - Looks up device code by user_code
   - Creates User in database if doesn't exist (with username and email)
   - Marks device code as verified and linked to user_id
   - This endpoint is called by the FRONTEND when user confirms in browser
   - HTTP 200 OK or 400 Bad Request

3. POST /api/auth/token
   - Request body: TokenPollSchema (device_code)
   - Returns TokenResponseSchema
   - If device code doesn't exist: 400 error
   - If device code exists but not verified: return {status: 202} (still waiting), HTTP 202
   - If device code verified: 
     - generate api_key
     - save to user
     - return {api_key, username, status: 200}, HTTP 200
   - This endpoint is polled by the CLI

SCAN UPLOAD ENDPOINT:

4. POST /api/scans/upload
   - Request body: ScanReportSchema (the scan report from CLI)
   - Header: Authorization: Bearer <api_key>
   - Returns {"scan_id": 123, "status": "created"}
   - HTTP 201 Created
   - Extract api_key from Authorization header using get_current_user dependency
   - Find user by api_key (if not found: 401 error)
   - Create Scan record in database for this user
   - Create Finding records for each finding in the report
   - Save all to database and commit
   - Return scan_id

SCAN RETRIEVAL ENDPOINTS:

5. GET /api/scans
   - Header: Authorization: Bearer <api_key>
   - Returns: list of ScanResponseSchema
   - HTTP 200 OK
   - Extract api_key and get user using get_current_user dependency
   - Query all Scans for this user, ordered by created_at DESC (newest first)
   - Return as array of ScanResponseSchema

6. GET /api/scans/{scan_id}
   - Header: Authorization: Bearer <api_key>
   - Path parameter: scan_id (integer)
   - Returns: single ScanResponseSchema with all findings
   - HTTP 200 OK if found, 404 if not found
   - Extract api_key and get user using get_current_user dependency
   - Query Scan by ID where user_id matches
   - If not found: 404 error
   - Include all findings for that scan (via relationship)
   - Return scan with findings

HELPER DEPENDENCY:

7. get_current_user(authorization: str = Header(None), db: Session)
   - Dependency that all protected endpoints use
   - Extract api_key from "Bearer <api_key>" format
   - If Authorization header missing or wrong format: raise 401 Unauthorized
   - Call get_user_by_api_key()
   - If not found: raise 401 Unauthorized
   - Return user object

ERROR HANDLING:
- 400: Invalid request (missing required fields, invalid format)
- 401: Unauthorized (missing/invalid API key)
- 404: Not found (scan doesn't exist)
- 500: Server error (unexpected exception)

CORS CONFIGURATION:
- Use CORSMiddleware from fastapi.middleware.cors
- Allow all origins for now (*, ), restrict later to basilisk.app domain only
- Allow all methods and headers

STARTUP:
- Create all tables with database.Base.metadata.create_all(engine)
- Log that server is running
